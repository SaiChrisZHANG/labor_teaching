# First Sprint

## Objective

Establish the repository conventions and ship one complete showcase module.

## Sprint order

### Step 1
Scaffold the repository from `docs/repo-blueprint.md`.

### Step 2
Create the shared toolkit shells:
- labor market facts and datasets
- identification primer
- reproducibility workflow

### Step 3
Build Labor I Module 1:
- learning objectives
- full note
- reading ladder
- exercises

### Step 4
Build the matching Beamer deck.

### Step 5
Build one Python code lab.

### Step 6
Add optional R and Stata sidecars only if they add real pedagogical value.

### Step 7
Run a cleanup pass:
- links
- headings
- naming
- bibliography placeholders
- repo landing page updates

## Review standard

At the end of the sprint, the repository should contain:
- a coherent folder structure
- one full teaching module
- one full slide deck
- one executable lab
- clear conventions Codex can reuse in later tasks
