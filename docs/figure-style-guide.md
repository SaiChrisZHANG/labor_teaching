# Figure Style Guide

This guide defines the Phase 1 house style for script-generated figures in Labor I and Labor II.

## Scope

Phase 1 applies only to figures generated by the audited Python scripts under:

- `books/labor-i/assets/scripts/`
- `books/labor-ii/assets/scripts/`

It does not redraw figures that lack a matching source script.

## Visual direction

- white background
- dark readable text
- calm, publication-style composition
- restrained palette with muted blues, teals, rusts, golds, and slate neutrals
- one shared sequential heatmap scale across both books

## Typography

- use one consistent sans-serif family across all script-generated figures
- keep title weight semibold rather than heavy slide-style bold
- use sentence case titles
- keep annotation text smaller than axis labels
- avoid long in-figure title blocks when the chapter caption already carries explanation

## Chart rules

- use thin, light grids only when they help reading values or trends
- remove top and right spines
- keep legends simple and frameless
- use consistent line widths and marker sizes
- prefer direct annotation only when it reduces legend clutter
- keep text off the data whenever possible

## Diagram rules

- align boxes to a visible grid
- use consistent rounded-box geometry and padding
- keep arrows outside labels and away from box text
- use dark neutral arrows unless a margin or mechanism genuinely needs color emphasis
- leave generous whitespace between conceptual clusters
- prefer short labels inside boxes and put longer interpretation in a subtitle or footer note

## Color rules

- `navy` is the default primary series color
- `rust` is the default contrast series color
- `teal` is the default secondary mechanism or adjustment color
- `gold` is used sparingly for emphasis, partitions, or phase shading
- `slate` is reserved for tertiary series or residual objects
- heatmaps should use the shared sequential map from `shared/figure_style.py`

## Layout rules

- use figure sizes that match the content type:
  - compact single-panel chart
  - wide two-panel comparison
  - diagram canvas with explicit bounds
- keep outer margins balanced
- avoid clipped text and titles
- prefer smaller, cleaner footer notes rather than dense explanatory text embedded in the middle of the figure

## Implementation notes

- the shared helper is `shared/figure_style.py`
- scripts should call the shared style setup before plotting
- scripts should use the shared save helper so output DPI, facecolor, and export behavior are consistent
- scripts should reuse shared axis and diagram helpers where appropriate

## Phase 1 exclusions

- no-source figures are tracked in `docs/figure-manual-followup.md`
- `books/labor-i/assets/scripts/generate_week1_facts.py` is excluded from regeneration in Phase 1 because running it would refresh live FRED data rather than perform a pure visual restyle unless data are frozen locally first
